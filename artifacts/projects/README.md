# Artifacts Projects

This folder is the always-accessible location for generated project artifacts.

## Purpose

- Store durable outputs that need to be reused across sessions.
- Keep project deliverables in one predictable path.

## Path

- `artifacts/projects/`

## Conventions

- Create one subfolder per initiative (example: `artifacts/projects/acheevy-digital/`).
- Prefer timestamped filenames for generated exports (example: `2026-03-23-routing-audit.md`).
- Do not store API keys, tokens, or secrets in this directory.

## Typical Contents

- Runbooks and decision logs.
- Exported reports and research packs.
- Validation outputs and handoff bundles.
