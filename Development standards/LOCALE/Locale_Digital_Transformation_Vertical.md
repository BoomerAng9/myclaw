## STORAGE & FILE TRANSFER DECISION: GCP VS. FIREBASE

### Executive Summary (Decision Framework)

**Question:** For Digital Transformation vertical, should we use **Google Cloud Storage (GCS)** or **Firebase Storage** for document vault?

**Answer:** **Hybrid Approach** (GCS for primary storage, Firebase for metadata + access control)

### Detailed Analysis

#### Option 1: Firebase Storage (Cloud Storage for Firebase)

**Pros:**
- ✅ Built on GCS (same backend, no compromise)
- ✅ Integrated with Firebase Auth (user identity flow automatic)
- ✅ Easy client-side uploads (web SDK handles auth + uploads)
- ✅ Firestore rules (access control via Firestore doc rules)
- ✅ Simpler architecture (one service, not two)

**Cons:**
- ❌ Limited lifecycle management (can't easily auto-delete old versions)
- ❌ No granular access control (rules are all-or-nothing)
- ❌ No advanced search (no metadata indexing built-in)
- ❌ No version tracking (Firebase doesn't track file versions natively)

**Cost:** $0.018/GB/month (same as GCS, just different pricing model)

---

#### Option 2: Google Cloud Storage (GCS) + Firestore Metadata

**Pros:**
- ✅ Fine-grained access control (bucket policies, signed URLs with expiry)
- ✅ Lifecycle management (auto-delete old versions, archive after N days)
- ✅ Version tracking (GCS object versioning native)
- ✅ Advanced features (retention policies, holds, compliance)
- ✅ Audit logging (Cloud Audit Logs, integration with SIEM)
- ✅ GDPR/HIPAA/SOC2 ready (built-in compliance controls)
- ✅ Search optimization (metadata in Firestore, full-text search via Algolia/Elasticsearch)

**Cons:**
- ❌ More complex auth (requires service accounts + signed URLs)
- ❌ Two services to manage (GCS + Firestore)
- ❌ Higher latency for metadata lookups (two DB queries, not one)

**Cost:** $0.020/GB/month (slightly higher than Firebase, worth it)

---

#### Option 3: Firebase Storage + Firestore Metadata (Hybrid, Recommended)

**The Best of Both Worlds:**

**Architecture:**
```
User uploads file
  ↓
Firebase Storage (via SDK, automatic auth)
  ↓
File stored in GCS backend
  ↓
Cloud Function triggered (on file upload)
  ↓
Creates Firestore metadata doc (filename, size, OCR status, user, created_at, access_log)
  ↓
Metadata Firestore doc (searchable, queryable, audit-logged)
```

**Pros:**
- ✅ Simple client upload (Firebase SDK)
- ✅ Granular access control (Firestore rules on metadata)
- ✅ Version tracking (metadata tracks old versions)
- ✅ Full-text search (Firestore metadata + Algolia)
- ✅ Audit logging (Firestore audit trails)
- ✅ GDPR/HIPAA compliance (metadata in Firestore, encryption at rest)
- ✅ Cost-effective (single storage layer, minimal duplication)
- ✅ Scalable (handles millions of documents)

**Cons:**
- ⚠️ Slightly more complex (needs Cloud Functions to sync metadata)
- ⚠️ Eventual consistency (metadata lag < 1 second, usually instant)

**Cost:** $0.018/GB/month (Firebase) + $0.18/100K reads (Firestore metadata) = ~$0.025/GB effective

#### Implementation Details

```typescript
// lib/storage.ts
import { getStorage, ref, uploadBytes } from "firebase/storage";
import { getFirestore, doc, setDoc } from "firebase/firestore";
import { v4 as uuidv4 } from "uuid";

export async function uploadDocument(
  file: File,
  userId: string,
  metadata?: { title?: string; category?: string }
) {
  const storage = getStorage();
  const db = getFirestore();
  
  // 1. Upload file to GCS (via Firebase Storage)
  const documentId = uuidv4();
  const storageRef = ref(storage, `documents/${userId}/${documentId}`);
  
  const snapshot = await uploadBytes(storageRef, file, {
    customMetadata: {
      uploadedBy: userId,
      originalName: file.name,
    },
  });
  
  // 2. Create metadata doc in Firestore
  const metadataRef = doc(db, "documents", documentId);
  await setDoc(metadataRef, {
    userId,
    documentId,
    filename: file.name,
    fileSize: file.size,
    mimeType: file.type,
    gcsPath: snapshot.metadata.fullPath,
    uploadedAt: new Date(),
    title: metadata?.title || file.name,
    category: metadata?.category || "uncategorized",
    ocrStatus: "pending", // Cloud Function will update this
    extractedData: null, // Will be populated by OCR
    accessLog: [
      {
        userId,
        action: "upload",
        timestamp: new Date(),
      },
    ],
    shareableLinks: [],
    securityLevel: "private",
    allowedViewers: [userId],
  });
  
  // 3. Cloud Function will automatically:
  // - Call Google Vision API for OCR
  // - Update ocrStatus, ocrText, extractedData
  // - Trigger indexing for full-text search
  
  return documentId;
}

// Download (with access control)
export async function downloadDocument(documentId: string, userId: string) {
  const db = getFirestore();
  const storage = getStorage();
  
  // 1. Check access (Firestore security rules)
  const docRef = doc(db, "documents", documentId);
  const docSnap = await getDoc(docRef);
  
  if (!docSnap.exists()) throw new Error("Document not found");
  
  const data = docSnap.data();
  if (!data.allowedViewers.includes(userId) && data.userId !== userId) {
    throw new Error("Access denied");
  }
  
  // 2. Log access
  await updateDoc(docRef, {
    accessLog: arrayUnion({
      userId,
      action: "download",
      timestamp: new Date(),
    }),
  });
  
  // 3. Get file from GCS
  const fileRef = ref(storage, data.gcsPath);
  const url = await getDownloadURL(fileRef);
  
  return url;
}

// Share via expiring link
export async function createShareableLink(
  documentId: string,
  userId: string,
  expiresInHours: number = 24
) {
  const db = getFirestore();
  const storage = getStorage();
  
  // Generate token
  const token = uuidv4();
  const expiresAt = new Date();
  expiresAt.setHours(expiresAt.getHours() + expiresInHours);
  
  // Store in metadata
  const docRef = doc(db, "documents", documentId);
  await updateDoc(docRef, {
    shareableLinks: arrayUnion({
      token,
      createdBy: userId,
      createdAt: new Date(),
      expiresAt,
    }),
  });
  
  // Generate shareable URL
  const shareUrl = `${window.location.origin}/share/${documentId}/${token}`;
  
  return shareUrl;
}
```
