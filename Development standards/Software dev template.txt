# Universal Build Tracking Template |03/16/[10:18 AM]

This template is built around the lifecycle developers are usually expected to follow: define requirements, design the system, implement, verify, secure, deploy, and maintain. I grounded it in NIST’s SSDF for secure development, OWASP ASVS for web app verification, the OpenAPI Specification for HTTP APIs, the MCP spec’s host-client-server model and version negotiation, Chrome Extension Manifest V3 requirements, and Core Web Vitals for web performance. ([NIST Computer Security Resource Center][1])

Use this as the master tracker for any product build: SaaS, MCP, API, AI app, internal tool, Chrome extension, or mixed platform.

## How to use it

Start every build by filling sections 0 through 4 before coding. During implementation, update sections 5 through 16 weekly. Do not mark section 17 as pass until the core user journey, security checks, and deployment checks are actually complete. That follows the same logic used by NIST SSDF and OWASP ASVS: security and verification are not separate afterthoughts. ([NIST Computer Security Resource Center][1])

## Criteria that make this template fit your stack

It covers:

* general software development lifecycle
* web app and GUI work
* SaaS billing and tenant logic
* API design and documentation through OpenAPI
* MCP products through host/client/server, transport, and version handling
* Chrome extensions through Manifest V3, service workers, permissions, and store prep
* AI/prompt systems through templates, evaluations, fallback rules, and safety
* post-launch maintenance and knowledge-index updates ([OpenAPI Initiative Publications][2])
